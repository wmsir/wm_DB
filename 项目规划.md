# wmDB V2.0 商业版 - 持续完善与项目规划说明

当前已完成核心框架的搭建，涵盖：Spring Security + JWT 实名认证、Druid AST 语法预检、长短文分流与 MinIO 流式处理、基础 MyBatis-Plus 模型与持久层配置，以及 Vite + Vue 3 包含 Monaco Editor 和动态防盗水印的前端。

为了将该项目打造为生产级可用的商业产品，后续应严格遵循以下开发路线进行完善：

## 1. 架构级强化规划
* **全局统一响应与异常处理** `[已完成]`：
  - 引入标准的 `Result<T>` 泛型统一包裹后端 API 返回值。
  - 配置 `@RestControllerAdvice` 拦截如 `MethodArgumentNotValidException`、`AccessDeniedException` 以及 Druid AST 自定义的语法解析异常。
* **安全加密升级** `[已完成]`：
  - 基础认证机制与 Token 防护 `[已完成]`。
  - 目前密码处理使用了简单的明文对比或 AES 脱敏。商业系统中必须在 `SysUser` 的 `password_cipher` 使用 `BCryptPasswordEncoder` `[已完成]`。
  - 前后端交互引入 RSA 非对称加密传输敏感凭证 `[已完成]`。
* **高可用缓存与速率限制** `[已完成]`：
  - 引入 Redis，用于 JWT 黑名单记录和登录状态追踪 `[已完成]`。
  - 实现基于 Redis 的全局 API 速率限制（防刷、防暴力破解）及 DBA 视图热点数据缓存 `[已完成]`。
* **统一日志采集与链路追踪** `[已完成]`：
  - 已实现 `SqlAuditLog` 异步入库记录审计 `[已完成]`。
  - 引入 ELK 或 SkyWalking 完善分布式链路 `[已完成]`。

## 2. 工作流 (Flowable) 深度集成
* **后端基础核心引擎挂载与执行防护** `[已完成]`
* **动态流程设计器** `[已完成]`：
  - 前端补充集成 `bpmn.js` 画布，允许管理员在页面动态拖拽连线，设计审核层级（如：一线主管 -> DBA -> 安全专员）。
* **自动化网关与服务任务** `[已完成]`：
  - 根据 `TicketService` 存储时评估的 `affectRowsEstimate`（影响行数）自动触发网关分支：小于 100 行自动通过（执行），大于 1w 行必须升级为多签审批。

## 3. 数据库引擎扩展与真实连接
* **AST 预检查与高危拦截 (Druid)** `[已完成]`
* **长短文智能分流机制 (MinIO Stream)** `[已完成]`
* **多数据源 (Dynamic DataSource)** `[已完成]`：
  - 接入 `dynamic-datasource-spring-boot-starter`。在 `DbInstance` 中存储的账号密码需要配置到内存态配置中心（如 Nacos）。在流式执行时，动态挂载对应数据源执行以防止主从错乱。
* **SPI 插件生态** `[已完成]`：
  - 已完成基于 SPI 规范的接口制定及 MySQL 原生能力 `[已完成]`。
  - 完善达梦数据库 (`DmJdbcDriver`) 和 Oracle 插件的扩展，实现各自方言的 `DbEnginePlugin` 预检校验逻辑 `[已完成]`。

* **基础交互界面与代码对比组件 (Monaco/Element-Plus)** `[已完成]`
* **动态防篡改水印防泄漏组件** `[已完成]`
* **状态管理与路由守卫** `[已完成]`：

* **UI 组件增强** `[已完成]`：

* **Docker 生产级编排** `[已完成]`：

  - 编写生产级 `docker-compose-prod.yml`，剥离外部存储卷、映射日志路径，并处理好时区与字符集。

## 6. 企业级权限系统对接与 RBAC 同步
* **外部组织架构同步对接** `[已完成]`：
  - 提供标准的 Webhook 接口接收外部企业现成的 OA/HR 系统的用户信息推送。
  - 支持 LDAP 或 OIDC 协议进行组织架构信息的定时拉取过滤。
* **动态角色映射与权限装配** `[已完成]`：
  - 将外部系统的部门或岗位特征，转换为本地 `SysUser` 的 `role` (ADMIN, DBA, DEV)。
  - 在 Flowable 审批配置中，基于映射后的本地角色动态分配 `candidateGroups` 候选人组，实现完全解耦的动态审批链。

## 7. 中国企业级规范与安全合规演进方向 (等保2.0/国密标准)
* **国密算法替换与改造 (SM2/SM3/SM4)** `[待实现]`：
  - 在信创与政企环境中，将密码哈希（BCrypt）替换为 SM3，非对称传输加密（RSA）替换为 SM2，数据库凭证对称加密（AES）替换为 SM4，满足《网络安全法》与等保三级强制要求。
* **信创国产化数据库深度适配** `[持续优化]`：
  - 现已支持达梦数据库（DM），后续需扩充人大金仓（Kingbase）、OceanBase、TiDB 以及高斯（openGauss）的专属 AST 校验与物理执行方言库。
* **审计日志抗抵赖与区块链存证** `[待完善]`：
  - 现有的 `SqlAuditLog` 虽设计了 `current_hash` 防篡改链，但在商业落地时应将 Hash 摘要定期上链（如长安链 ChainMaker）或与第三方公证处系统对接。
* **全局中文语境与错误码标准化 (I18n)** `[待优化]`：
  - 严格按照阿里 Java 开发规范，梳理全局异常抛出，返回统一的标准企业级错误码（如 `A0400`），避免直接将底层 SQLException 的英文堆栈抛给前台。

## 8. 商业化高阶演进与架构体验优化
* **智能读写分离与高危拦截** `[已完成]`：
  - **DML/DDL (变更)**：强制路由至主库执行。引入影子重放（Flashback）机制，自动解析并生成一键回滚语句。
  - **DQL (查询/报表)**：自动将纯查询请求路由至只读从库，避免拖垮主库性能（已实现 `isDqlOnly` AST 判断）。
* **自动化与智能化审核 (SQL Lint)** `[已完成]`：
  - 自动检测缺失索引等问题，执行前预估执行计划（已实现基于 EXPLAIN 拦截全表扫描 `type=ALL` 的策略）。
* **数据安全与合规防御** `[已完成]`：
  - **动态数据脱敏 (Data Masking)**：对线上查询结果中的敏感字段（如手机号、身份证、邮箱）自动打码屏蔽（已实现正则匹配屏蔽）。
* **高效协同与执行管理** `[已完成部分]`：
  - **IM 深度集成**：设计了企业级 IM 聚合 Webhook 推送。
  - **维护窗口定时执行**：支持 DBA 审批后，通过 `executionWindow` 字段放入等待队列延时执行。
